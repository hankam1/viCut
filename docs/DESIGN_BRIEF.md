# ViCut — дизайн-бриф

Бриф для проектирования фирменного стиля и интерфейса десктоп-приложения ViCut. Самодостаточный документ: его можно отдавать дизайнеру или Claude Design целиком.

---

## 1. О продукте

**ViCut** — опенсорсное десктоп-приложение (Windows + macOS) для автоматического монтажа видео. Пользователь один раз настраивает **пресет** (субтитры, переходы, громкость, эффекты), кидает файлы в **очередь** — и получает готовые видео без ручного монтажа. Вся тяжёлая работа делается локально (FFmpeg + Whisper), опционально — облачная транскрипция по API-ключу.

- **Аудитория:** ютуберы и создатели контента, которым нужно быстро и одинаково оформлять регулярные видео. Не профессиональные монтажёры — люди, которые хотят «кинул → получил».
- **Анти-цель:** мы НЕ таймлайн-редактор (не Premiere, не CapCut). Никаких дорожек и скраббинга. Ценность — в автоматизации и предсказуемости.
- **Конкурентное окружение по духу:** HandBrake (утилитарность), Descript (дружелюбие), Raycast/Linear (минимализм и скорость).

## 2. Характер бренда

| Ось | Позиция |
|---|---|
| Простой ↔ Мощный | Простой снаружи, мощный внутри |
| Игривый ↔ Серьёзный | Спокойный, уверенный, без клоунады |
| Тёплый ↔ Технологичный | Технологичный, но не холодный |

Три слова: **точный, тихий, быстрый**. Интерфейс не должен «кричать» — видео пользователя главнее интерфейса.

## 3. Логотип

### Основная концепция — «V-склейка» (V + playhead)

Знак строится из смысла продукта: **V = два клипа, сведённые в точку склейки**.

- Две наклонные широкие полосы (как два фрагмента плёнки/таймлайна) сходятся вниз, образуя букву **V**.
- Между ними — тонкая **вертикальная линия playhead** (курсор таймлайна), спускающаяся сверху точно в вершину V. Она же читается как линия разреза: место, где два клипа «склеились».
- Геометрия простая, из прямых линий со скруглёнными концами (round caps) — знак должен собираться из 3 штрихов.

```
  ╲   |   ╱      две полосы = клипы
   ╲  |  ╱       линия по центру = playhead / разрез
    ╲ | ╱
     ╲|╱         вершина = точка склейки
```

Требования к знаку:
- Читается в 16×16 (favicon/трей) и в 1024×1024 (иконка приложения).
- Работает в монохроме (один цвет) и в инверсии (тёмный/светлый фон).
- В цветной версии: полосы — градиент акцентного цвета, playhead — белый (на тёмном фоне).
- Иконка приложения: знак на скруглённом квадрате цвета фона (#0B0D10), в стиле macOS Big Sur+ / Windows 11.

### Wordmark

**ViCut** — гротеск средней жирности (semibold), обычный регистр, без курсива. Фишка: точка над **i** заменяется на маленький квадратный маркер акцентного цвета (метка «rec»/маркер таймлайна), а сама палочка «i» — это мини-playhead. Wordmark должен работать и без этой детали (fallback — чистый текст).

### Альтернативы (если основная не сработает)

1. **«Кадр-в-развороте»:** прямоугольник кадра 16:9, рассечённый диагональю с небольшим сдвигом половинок — момент склейки.
2. **Чистая монограмма:** геометричная V с оптическим вырезом-треугольником, отсылающим к кнопке Play, повернутой вершиной вниз.

## 4. Цвет

Тёмная тема — **первичная** (видео смотрят в темноте, приложение работает рядом с плеером). Светлая — вторичная, обязательна к поставке.

### Тёмная тема (основная)

| Токен | Значение | Использование |
|---|---|---|
| `bg` | `#0B0D10` | фон окна |
| `surface` | `#14171C` | карточки, панели |
| `surface-2` | `#1C2027` | ховеры, вложенные блоки |
| `border` | `#262B33` | разделители, обводки |
| `text` | `#E8EAED` | основной текст |
| `text-muted` | `#8B93A1` | вторичный текст, подписи |
| `accent` | `#7C5CFF` | электро-фиолет: кнопки, ссылки, активные состояния, прогресс |
| `accent-hover` | `#9070FF` | ховер акцента |
| `success` | `#3DDC84` | done, галочки |
| `warning` | `#FFB224` | предупреждения |
| `danger` | `#FF5C5C` | failed, деструктивные действия |

Правило: **один акцент на экран**. Прогресс рендера — главный «живой» элемент, он использует accent. Никаких радуг: статусы — только success/warning/danger.

### Светлая тема

Те же токены, значения: bg `#FAFBFC`, surface `#FFFFFF`, border `#E4E7EC`, text `#171A1F`, muted `#697180`. Акцент тот же `#7C5CFF` (проверить контраст на белом — при необходимости затемнить до `#6A4CE8`).

## 5. Типографика

- **UI:** Inter (или Manrope как альтернатива), 13–14px базовый размер, 400/500/600.
- **Числа и таймкоды:** те же шрифты с `font-variant-numeric: tabular-nums` — прогресс и таймкоды не должны «прыгать». Для длинных логов — JetBrains Mono.
- Заголовки страниц: 18–20px semibold. Никаких декоративных шрифтов.

## 6. Иконки и иллюстрации

- Иконки: **Lucide**, stroke 1.5px, размер 16/20px. Не смешивать наборы.
- Пустые состояния: минималистичные line-art иллюстрации в тех же stroke-правилах, один акцентный цвет. Без 3D-персонажей и стоковых улыбок.

## 7. Структура приложения и экраны

Окно: минимум 1100×720, resizable. Навигация — узкий левый сайдбар с иконками (4 пункта): **Очередь**, **Пресеты**, **История** (пост-MVP), **Настройки**. Кастомный тайтлбар в цвет фона (frameless).

### 7.1 Очередь — главный экран

- **Hero-зона drag&drop** сверху: пунктирная рамка «Перетащи видео сюда или нажми, чтобы выбрать». Всегда видна, сжимается, когда в очереди есть задачи.
- **Список задач-карточек.** Каждая карточка: статус-индикатор, название, пресет (бейдж), миниатюра первого кадра, кол-во клипов, длительность → и справа статус:
  - `pending` — тихий серый, кнопка «отменить»;
  - `running` — прогресс-бар (accent) + текущая стадия словами: «Анализ → Аудио → Субтитры → Кодирование», скорость (например «12.4× realtime»);
  - `done` — зелёная галочка, кнопки «Открыть» / «Показать в папке»;
  - `failed` — красный, раскрываемая ошибка человеческим языком, кнопка «Повторить».
- Нижняя панель очереди: «N задач · выполняется #2» + большая кнопка **Запустить очередь / Пауза**.
- Пустое состояние: иллюстрация + «Очередь пуста. Перетащи первое видео».

### 7.2 Мастер новой задачи (после drop)

Модальный лист в 2 шага, не страница:
1. **Файлы:** список брошенных клипов, drag-сортировка порядка склейки, суммарная длительность.
2. **Пресет и выход:** горизонтальные карточки пресетов (имя + 3 иконки-подсказки: субтитры/переход/громкость), поле папки вывода, имя файла. Кнопка **В очередь**.

Принцип: от drop до задачи в очереди — максимум 2 клика.

### 7.3 Редактор пресетов

Двухколоночный: слева список пресетов (встроенные помечены замком — редактирование через «Дублировать»), справа — настройки выбранного, сгруппированные в секции-аккордеоны:

- **Выход:** качество (high/medium/low — segmented control), кодек, разрешение, FPS.
- **Звук:** тумблер нормализации + слайдер LUFS (с подписью «−14 — стандарт YouTube»).
- **Субтитры:** тумблер; провайдер (auto/локально/Groq/OpenAI); язык; стиль — **живой превью-блок**: тёмный кадр 16:9 с примером фразы, обновляется при изменении шрифта/размера/цвета/позиции; переключатели «вшить в видео» и «сохранить .srt».
- **Переходы:** сетка мини-превью типов переходов (анимированные на ховер), слайдер длительности.
- **Эффекты:** слайдеры яркость/контраст/насыщенность/резкость со значением по центру и кнопкой сброса, выбор LUT-файла.

Изменения сохраняются сразу (autosave + toast «Сохранено»), у встроенных пресетов — кнопка «Дублировать и редактировать».

### 7.4 Настройки

- **Транскрипция:** статус локального Whisper (модель, «скачать/обновить», занятое место), поля API-ключей Groq/OpenAI (masked, «вставить и проверить»).
- **Инструменты:** статус FFmpeg (версия, путь), кнопка перекачать.
- **Общее:** тема (тёмная/светлая/системная), язык интерфейса (RU/EN), папка вывода по умолчанию.

### 7.5 Первый запуск (onboarding)

Один экран, не карусель: логотип, одна фраза («Монтаж по пресетам. Кидаешь видео — получаешь готовое.»), прогресс скачивания FFmpeg/Whisper с человеческими подписями («Скачиваем FFmpeg — 96 МБ»), кнопка «Начать» активируется по готовности. Опция «пропустить Whisper — субтитры настроишь позже».

## 8. Ключевые компоненты (дизайн-система)

`Button` (primary/secondary/ghost/danger) · `Card` задачи · `Badge` статуса и пресета · `StageProgress` (стадии рендера с текущей активной) · `ProgressBar` (determinate + indeterminate) · `SegmentedControl` · `Slider` со значением · `Toggle` · `Select` · `Dropzone` · `Toast` · `Modal/Sheet` · `EmptyState` · `SubtitlePreview` (кадр с живым стилем субтитров).

## 9. Движение

- Длительности: 120–200 мс, easing `ease-out`. Никаких bounce.
- Прогресс-бары — плавная интерполяция, не скачки.
- Карточка задачи при завершении: короткая вспышка success-цвета по левой кромке.
- Появление задач в списке: fade + 4px slide-up.

## 10. Тексты (tone of voice)

Коротко, в настоящем времени, без жаргона и без восклицательных знаков. Ошибки — человеческим языком с действием: не «Error: exit code -22», а «Не удалось вшить субтитры. Попробуй ещё раз или отключи субтитры в пресете» (техподробности — под спойлером «Детали»). Локализация RU + EN с первого дня; писать так, чтобы строки не ломали вёрстку при переводе (+30% длины).

## 11. Доступность

Контраст текста ≥ 4.5:1; видимые focus-кольца (accent, 2px offset); полная навигация с клавиатуры (очередь, формы); статусы не только цветом, но и иконкой/текстом; поддержка `prefers-reduced-motion`.

## 12. Технический контекст

Реализация: **Electron + React + Tailwind CSS**. Дизайн должен быть выполним на веб-стеке без экзотики: системные шрифты — fallback, эффекты — CSS (без тяжёлых blur-слоёв на всё окно). Иконка приложения нужна в .ico (Win) и .icns (macOS).

## 13. Ожидаемые результаты (deliverables)

1. **Логотип:** знак + wordmark + иконка приложения (SVG-мастер; экспорт 16–1024px), монохром и цвет, тёмный/светлый.
2. **Дизайн-токены:** цвета, типографика, отступы, радиусы, тени — в JSON/CSS-переменных.
3. **Макеты экранов** (dark first, потом light): Очередь (все 4 состояния карточек + пустое), Мастер новой задачи (2 шага), Редактор пресетов (с превью субтитров), Настройки, Онбординг.
4. **Спецификация компонентов** из раздела 8 со состояниями (default/hover/active/disabled/focus).
